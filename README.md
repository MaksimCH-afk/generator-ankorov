# Генератор URL-анкоров с процентным распределением

Веб-инструмент для SEO: принимает список доменов и частотки ключей, рассчитывает
план размещения ссылок по типам анкоров (в процентах согласно выбранной стратегии)
и выгружает готовый **Excel**. Реализация по ТЗ v0.3.

## Возможности

- **Стратегии (профили анкоров)** — безанкор + роли ключей (`основной 1/2`, `добавочный 1/2…`).
  Базовые «Обычная» (70/12/9/5/4) и «Безопасная» (75/13/5/4/3). Можно создавать
  кастомные с произвольным числом добавочных ролей. Валидация суммы = 100% (§3.3, §9).
- **Типы кампаний** → отдельные вкладки в Excel: «Прогоны» (полный профиль),
  «Крауд+сабмиты» (100% безанкор), «Внутренние страницы».
- **Безанкорные профили** — сохранённые наборы форматов (как стратегии): голый
  URL (`https://site.com/`) и голый домен (`site.com`) в нужной пропорции.
  3 базовых профиля + создание своих, с живым примером для каждого.
- **Внутренние страницы** — справочник `тип страницы + язык → суффикс`
  (en/de/pl/tr/pt-br); анкор = `{бренд} {суффикс}`, по одной строке на страницу.
- **Импорт проектов из Excel** — один файл, где каждая вкладка = проект (ключи +
  частотность + домен). Колонки распознаются по синонимам в любом порядке, KD
  игнорируется, домен находится автоматически (`http→https`), пустые и сводные
  листы пропускаются. Проекты создаются/обновляются по домену.
- **Шутки про SEO** — виджет в шапке обновляется при каждом переходе
  (OpenRouter, с локальным фолбэком).
- **Перераспределение отсутствующих ролей** — по умолчанию освободившийся % уходит в
  безанкор; есть ручная корректировка через JSON (§4.2).
- **Выгрузка** — один домен = один Excel с вкладками; для нескольких — выбор
  ZIP-архив / раздельные файлы (§6, §11). Предпросмотр результата перед скачиванием.
- **Язык статьи** — выпадающий список предустановленных языков; опция «Не указывать
  в выгрузке» полностью убирает колонку Article Language из Excel.
- **История** — каждая генерация сохраняется в БД (проект, стратегия, объёмы,
  формат выгрузки, число ссылок по вкладкам).
- **Логи** — детальный журнал событий (загрузки, правки, генерации, ошибки) с
  фильтрами по уровню и категории.
- **Справка на каждой странице** — блок-подсказка внизу для быстрого старта.
- **Детерминированность** и контроль суммы: количество строк на URL всегда = объёму (§9).

## Технологии

FastAPI · SQLAlchemy + SQLite · Jinja2 · openpyxl · Docker.

## Запуск через Docker (рекомендуется)

```bash
docker compose up --build
```

Откройте **http://localhost:9999**. БД SQLite хранится в volume `anchor_data`
(`/data/app.db`) и переживает перезапуски.

### Ключ для шуток про SEO (необязательно)

Виджет шуток работает и без ключа (на встроенном списке). Чтобы шутки
генерировались моделью OpenRouter — создайте файл `.env` рядом с
`docker-compose.yml`:

```bash
cp .env.example .env
# впишите OPENROUTER_API_KEY=sk-or-...
```

`.env` в репозиторий не попадает (в `.gitignore`).

## Локальный запуск без Docker

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 9999
```

## Тесты

```bash
pip install pytest
pytest tests/ -q
```

Тесты покрывают раскладку ролей по частотности, контроль суммы = объёму, поглощение
остатка округления безанкором, перераспределение ролей, дробление по форматам,
внутренние страницы и детерминированность.

## Быстрый сценарий использования

1. **Стратегии** — проверьте/создайте профиль анкоров.
2. **Безанкорные форматы** — настройте дробление безанкора (по желанию).
3. **Внутренние страницы** — отредактируйте справочник суффиксов по языкам.
4. **Проекты** — создайте проект (URL, язык, бренд), загрузите частотку,
   выберите стратегию, задайте объёмы «Прогоны»/«Крауд», укажите внутренние страницы.
5. **Генерация** — выберите проекты, формат выгрузки (ZIP/раздельные) → скачайте Excel.

## Структура

```
app/
  main.py          # FastAPI: дашборд + эндпоинты
  generator.py     # чистая логика генерации (§4) — без I/O, тестируемая
  service.py       # мост ORM ↔ генератор
  excel_export.py  # сборка .xlsx и ZIP
  parsing.py       # парсинг CSV/Excel частоток
  models.py        # ORM-модели (SQLite)
  database.py      # подключение к БД
  seed.py          # базовые стратегии/форматы/суффиксы
  templates/       # Jinja2-шаблоны дашборда
  static/          # CSS
tests/             # юнит-тесты логики генерации
Dockerfile · docker-compose.yml · requirements.txt
```

## Колонки выходного Excel (§6)

| Link Q-ty | URL | Anchor | Article Language | Keyword |
|---|---|---|---|---|

Для безанкорных строк `Anchor` и `Keyword` равны заданному безанкорному формату
(`https://site.com/`, markdown и т.п.).
